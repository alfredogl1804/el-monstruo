## Nombre
**Nombre principal:** Vigilia Sincrónica — Monstruo Multinúcleo.
**Alias histórico:** Vigilia Mesh (nota: "mesh" libre es peligroso, la topología es coreografía radial).

## Tesis
El Monstruo no está vivo porque un hilo nunca duerme; está vivo porque múltiples loops finitos se relevan sin perder estado detrás de un único Monstruo visible.

## Fórmula Corregida
Reactor de Vigilia = 
- Unified Face
- + Rotor/Dispatcher
- + State Fabric soberano single-writer
- + Event Log append-only
- + Loops especializados de corta vida
- + Handoff / Restore Protocol
- + T1 humano

## Frase Doctrinal
"Un solo rostro, muchas mentes, una memoria soberana, una autoridad humana. La malla simula continuidad para el usuario, pero nunca simula autoridad ante T1."

## Diferenciación
- **Mesh libre:** Peligroso, riesgo de Dory distribuido o loops negociando autoridad.
- **Vigilia Sincrónica:** Coreografía radial con un dispatcher central.
- **Vida ininterrumpida:** Es una metáfora UX/operativa (precisión suiza), no una conciencia filosófica.
- **State Fabric:** Fuente soberana de estado, no un chat compartido.
