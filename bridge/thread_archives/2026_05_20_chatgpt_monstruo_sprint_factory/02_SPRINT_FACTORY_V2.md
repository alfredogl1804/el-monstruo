# 02 SPRINT FACTORY V2

**Estado:** DOCTRINE_CANDIDATE
**Fuente:** assistant_synthesis, t1_decision

## Concepto Core
Transición de un modelo de restricciones universales rígidas a un modelo de "permisos positivos" encapsulados por sprint.

## Pipeline Operativo
1. **Sprint Batch:** Definición de un lote de sprints.
2. **T1 Signoff:** Aprobación explícita de Alfredo (T1) sobre el alcance y permisos.
3. **Parallel Execution:** Ejecución concurrente por múltiples agentes (Manus, Cowork, etc.).
4. **Evidence Pack:** Generación de evidencia auditable al finalizar.
5. **Close / Next Batch:** Cierre formal y transición al siguiente lote.

## Permisos Positivos
En lugar de decir "Manus nunca puede tocar main", se define por sprint:
- "En el Sprint X, Manus tiene permiso para tocar main en la ruta Y".
- Las restricciones viven dentro del spec del sprint, no como reglas absolutas globales que paralizan la operación.
