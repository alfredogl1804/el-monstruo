## Propósito
Consolidar la explosión conceptual del 2026-05-20 (Vigilia Sincrónica, Reactor de Vigilia, Reactor Soberano, Escalera de Autonomía, SHELL) en una **stack vertical implementable**, eliminando redundancias y priorizando lo que se puede codificar hoy.

## La Stack Vertical (De Infraestructura a Optimización)

### Capa 1: Infraestructura Temporal (Vigilia Sincrónica)
**El problema:** Ningún hilo vive para siempre.
**La solución:** Coreografía radial de loops finitos detrás de un Unified Face.
**El mecanismo:** State Fabric single-writer + Event Log append-only + Loop Registry.
*Es la base física que permite que el Monstruo exista en el tiempo.*

### Capa 2: Comportamiento Autónomo (Reactor de Vigilia)
**El problema:** Si el Monstruo solo responde, no está vivo operativamente.
**La solución:** Tiempo propio y Dual Task Queue.
**El mecanismo:** Scheduler soberano que ejecuta tareas propias (Self-Evolution) cuando no hay tareas de usuario (User Mission).
*Es el motor que decide qué hacer con el tiempo que le da la Capa 1.*

### Capa 3: Permisos y Límites (Escalera de Autonomía)
**El problema:** ¿Cuánto puede hacer el Monstruo sin preguntar?
**La solución:** Escalera estricta A0-A8.
**El mecanismo:** Action Ladder integrada en el Loop Contract. Cada loop nace con un nivel de autonomía máximo permitido.
*Es la barra de control nuclear que hace segura a la Capa 2.*

### Capa 4: Economía y Vocabulario (Reactor Soberano)
**El problema:** Necesitamos un modelo mental para medir el éxito.
**La solución:** La metáfora de la Potencia Autónoma Útil (PAU).
**El mecanismo:** Absorber "Reactor Soberano" no como una pieza de software separada, sino como la *física/economía* del Reactor de Vigilia. Los embriones son las turbinas; el tiempo es el combustible.
*Es el marco de evaluación de las Capas 1 y 2.*

### Capa 5: Optimización de Canal (SHELL / Micropolvo Semántico)
**El problema:** El contexto largo es lento y costoso entre loops.
**La solución:** Canal IA↔IA multidimensional (Micropolvo Semántico).
**El mecanismo:** Parking lot de investigación (Research R0). No es ruta crítica para el MVP.
*Es una optimización futura para cuando la Capa 1 se vuelva el cuello de botella.*

## Conclusión Ejecutiva
Esta stack no son cinco ideas compitiendo. Es un solo sistema:
**El Monstruo corre sobre Vigilia Sincrónica (1), ejecutando el Reactor de Vigilia (2), gobernado por la Escalera de Autonomía (3), midiendo su éxito en PAU (4), y algún día se optimizará con SHELL (5).**
