## El Stack Seleccionado para Fase 1 (MVP)

De las 60 herramientas evaluadas, las siguientes forman el núcleo operativo inicial:

### Capa 1: Cerebros Fundacionales
- **Qwen 3.5**: Qwen 3.5 ofrece una relación costo-beneficio radicalmente superior a sus competidores, permitiendo escalar operaciones de IA de forma sostenible. Es ideal para un emprendedor individual.

### Capa 1: Cerebros Fundacionales (LLMs)
- **Grok 4.20 Multi-Agent Beta (Feb 17, 2026)**: Grok-4 es una adición estratégica indispensable para la Fase 1. Su acceso a datos en tiempo real del ecosistema Musk le otorga una ventaja única e inalcanzable para la competencia, resolviendo el problema del conocimiento desactualizado. Aunque su costo puede ser premium, su rol como 'sabio' especialista en actualidad justifica la inversión. La arquitectura multi-agente promete un razonamiento superior y se alinea con la filosofía del Monstruo. Su integración es de baja complejidad y es totalmente compatible con el stack actual. Adoptarlo es clave para dotar al sistema de una capacidad de análisis verdaderamente consciente del presente.

### Capa 2: Conectividad
- **MCP (Model Context Protocol)**: MCP es la apuesta estratégica correcta para la interoperabilidad de la IA en la Fase 1, alineando a El Monstruo con el estándar emergente de la industria y evitando deuda técnica futura.

### Capa 2: Conectividad (MCP & Tools)
- **Hasura**: Hasura es la elección pragmática para un emprendedor individual en Fase 1. Su capacidad para generar una API de datos GraphQL segura y performante en minutos es un acelerador inigualable.
- **n8n**: Se adopta n8n por ser una plataforma de automatización avanzada, open-source y auto-hospedable, ideal para los flujos complejos del Monstruo. Su flexibilidad y control de costos la hacen superior a alternativas para este caso de uso.

### Capa 4: Brazos Ejecutores
- **Claude Code v1.0.36**: Claude Code es la elección correcta para el rol de agente de codificación en Fase 1. Su capacidad para comprender el contexto completo de un proyecto y actuar de forma autónoma en tareas complejas (refactorización, PRs, tests) ofrece un salto cualitativo frente a simples autocompletadores como Copilot.
- **Cline**: Cline es un "Brazo Ejecutor" ideal para un desarrollador individual. Su modelo open-source, BYOK y el enfoque "human-in-the-loop" ofrecen un balance perfecto entre autonomía, costo y seguridad para la Fase 1.
- **Cursor IDE v0.48.7**: Cursor es la mejor herramienta en su clase para acelerar el desarrollo en Fase 1. Su arquitectura como IDE nativo de IA y su flexibilidad de modelos superan a la competencia, justificando la inversión.
- **Manus AI**: Se adopta como brazo ejecutor principal por su potencia y seguridad (CodeAct + Sandbox). Aunque es propietario y presenta riesgo de lock-in, su madurez es clave para el MVP. Se debe vigilar el costo.
- **OpenHands**: OpenHands es la elección ideal para la ejecución segura de agentes de software en Fase 1. Su arquitectura open-source con sandboxing en Docker es perfecta para un emprendedor individual, evitando complejidad innecesaria.

### Capa 4: Brazos Ejecutores (Agentes)
- **Gemini Deep Research Agent**: Se adopta como el agente especialista para tareas de investigación profunda. Su modelo de "planificar y luego ejecutar con aprobación" es ideal para tareas complejas y auditables. La arquitectura de 3 niveles de razonamiento ofrece un control de costos sin precedentes, vital para un emprendedor individual. Aunque carece de 'Function Calling' hoy, su rol como generador de inteligencia es tan valioso que justifica su inclusión, pasando sus resultados a otros agentes para la ejecución.

### Capa 4: Orquestación y Agentes
- **MultiOn**: Adopción clave. MultiOn no es otro orquestador, es el brazo ejecutor que permite a los agentes actuar en cualquier web a nivel de UI, un vacío crítico en el stack actual. Es sinérgico, no competitivo.

### Capa 5: Entorno de Ejecución (Runtime / Agent Fabric)
- **E2B**: E2B es la solución ideal para la ejecución segura de código de agentes en Fase 1. Ofrece sandboxing a nivel de hardware con la simplicidad de un SaaS, siendo la forma más rápida y segura de habilitar la ejecución de código para un desarrollador individual.
- **Modal**: Modal es la pieza perfecta para la Capa 5. Su simplicidad, modelo pay-as-you-go y el free tier lo hacen ideal para un MVP, resolviendo el problema crítico de dónde ejecutar tareas de IA pesadas sin gestionar servidores.

### Capa 5: Entorno de Ejecución (Runtime)
- **Docker**: Docker es el estándar de facto para la contenerización. Su simplicidad, madurez y ecosistema lo hacen una pieza fundamental e innegociable para la Capa 5 del Monstruo en Fase 1.

### Capa 6: Economía
- **LiteLLM**: LiteLLM es la elección definitiva y pragmática para el Gateway de LLMs en Fase 1. Como solución open-source y auto-hospedada, otorga control total sobre la privacidad de los datos y los costes, alineándose perfectamente con el contexto de un desarrollador individual.

### Capa 6: Orquestación y Economía (Agentic Market)
- **LangGraph**: LangGraph se adopta como la columna vertebral de orquestación para El Monstruo v2.0. A pesar de su complejidad media-alta, su paradigma de máquina de estados y control explícito son requisitos no negociables para construir un sistema de agentes fiable y auditable desde la Fase 1.

### Capa 7: Validación y Consenso
- **DSPy**: DSPy es una apuesta estratégica correcta. Su enfoque en optimización programática ataca la raíz del problema de la fragilidad de los prompts. Para un MVP de Fase 1, su capacidad para hacer que los LLMs rindan más con menos esfuerzo manual es crucial. Se adopta como el estándar para optimizar las tareas de los agentes orquestados por LangGraph.
- **Langfuse**: Se adopta Langfuse como la plataforma de observabilidad para la Fase 1. Su naturaleza open-source y el control sobre los datos superan la carga de mantenimiento del auto-alojamiento para un operador individual.
- **promptfoo**: Se adopta por ser un framework de evaluación TDD simple, open-source y local-first, ideal para un dev individual. Asegura la calidad de prompts y previene regresiones, integrándose perfectamente al stack.
