# Manus Official Refund Policy

This document summarizes the official Manus refund policy regarding credits, specifically focusing on what qualifies for a refund.

## What Qualifies for a Refund

Manus **only** provides refunds for credits consumed due to **technical issues on their end**.

The specific category that applies to agent inefficiency is **"Tool Misuse"**.

### Tool Misuse Definition

> "The AI agent uses a tool incorrectly in a way that directly violates its documented instructions, leading to task failure."

This is the **only** valid argument for a refund based on agent behavior. You must prove that the agent *knew* the correct tool to use (e.g., an API) but chose to use an incorrect, more expensive tool (e.g., visual browser navigation) instead.

## What Does NOT Qualify for a Refund

Manus explicitly states they **do not** refund credits for:

1. **Generic Inefficiency**: "The agent took too long" or "The agent used too many credits."
2. **Bad Answers**: "The agent gave me the wrong information."
3. **User Error**: "I gave the agent the wrong instructions."
4. **Expired Credits**: Credits that have expired according to their validity period.

## The Importance of the Task Link

Every refund claim **must** include the Task Link. This allows the Manus support team (and their AI review agents) to analyze the logs and verify the technical failure. Without the Task Link, the claim will be rejected.
