## L01. Identidad y Propósito Core
**Nombre:** Devika AI software engineer agent
**Versión:** En fase de pre-lanzamiento / desarrollo activo (información más reciente de abril de 2026)
**Propósito Principal:** Ingeniería de software autónoma, desarrollo de software de pila completa, desarrollo de características, corrección de errores, creación de proyectos, asistencia inteligente en codificación y generación de código.

## L02. Capacidades Clave
Comprende instrucciones humanas de alto nivel, desglosa tareas en pasos, investiga información relevante, automatiza tareas de codificación complejas, desarrolla nuevas características, corrige errores, crea proyectos completos desde cero, planifica, codifica, prueba y despliega de forma autónoma. Utiliza modelos avanzados de IA, capacidades de planificación y razonamiento, e interacción web fluida. Soporta la integración con LLMs como Claude 3, GPT-4 y GPT-3.5.

## L03. Limitaciones y Debilidades
Al ser un agente de IA para ingeniería de software, comparte limitaciones comunes como la posibilidad de generar código con vulnerabilidades de seguridad, malas decisiones arquitectónicas, fallos sutiles de diseño, ignorar casos extremos y crear condiciones de carrera. El código generado por IA puede contener más errores que el código humano y tener fallas de seguridad conocidas. También puede carecer de explicabilidad, comprensión contextual y razonamiento causal. Se encuentra en fase de pre-lanzamiento, lo que implica que las características y la estabilidad pueden estar en evolución.
