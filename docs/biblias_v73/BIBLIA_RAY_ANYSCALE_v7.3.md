<td>Precio Actual</td>
<td>Anyscale opera bajo un modelo de pago por uso, sin tarifas mensuales fijas, con descuentos por volumen a medida que aumenta el uso. Ofrecen contratos comprometidos para descuentos por volumen y otros beneficios. Los costos de cómputo se basan en el tipo de instancia, con tarifas por hora (AC /hr) para CPU y varias GPUs (NVIDIA T4, L4, A10G, A100, H100, H200). Por ejemplo, una instancia solo con CPU cuesta AC 0.0135 /hr, mientras que una con NVIDIA H200 cuesta AC 10.6812 /hr. Se ofrece un crédito inicial de $100 en Anyscale Credits para nuevos usuarios.</td>
</tr>
<tr>
<td>Posicionamiento Competitivo</td>
<td>Ray (Anyscale) se posiciona como una plataforma unificada y de código abierto para escalar aplicaciones de IA y Python, ofreciendo una alternativa a soluciones como Databricks para cargas de trabajo de aprendizaje automático escalables. Su diferenciador clave es la capacidad de escalar el mismo código desde una laptop hasta un clúster, con un enfoque en la optimización de la utilización de GPU y la reducción de costos. Anyscale complementa a Ray con una plataforma gestionada que ofrece herramientas de desarrollo mejoradas, escalado estable, costos reducidos y soporte experto, lo que la hace atractiva para la implementación de IA en producción a gran escala.</td>

<td>Cambios Clave desde Marzo 2026</td>
<td>Desde el 20 de marzo de 2026, Ray (Anyscale) ha introducido varias actualizaciones significativas. En abril, se lanzó Ray 2.55.1 (22 de abril) y Ray 2.55.0 (13 de abril) como imágenes base de Anyscale. Se añadió soporte para la tolerancia a fallos del nodo principal en recursos de la nube respaldados por Kubernetes (29 de abril) y se redujo la sobre-suscripción de memoria en clústeres de Ray (29 de abril). Anyscale ahora ofrece 'agent skills' para Claude Code y Cursor (22 de abril). Se actualizó el soporte para nuevos tipos de instancias en AWS y Google Cloud (7 de abril). En marzo, se lanzó Ray 2.54.1 (25 de marzo) y se introdujeron configuraciones de cómputo declarativas en la consola de Anyscale (23 de marzo). El operador de Anyscale para Kubernetes recibió actualizaciones a las versiones 1.5.0 (18 de marzo) y 1.5.1 (15 de abril), añadiendo soporte para Gateway API y corrigiendo errores. Múltiples versiones del Anyscale CLI y SDK fueron lanzadas a lo largo de marzo y abril, siendo la más reciente la 0.26.99 (29 de abril). También se emitió una advertencia de seguridad sobre la vulnerabilidad CVE-2026-31431 ("CopyFail") en el kernel de Linux (30 de abril).</td>
